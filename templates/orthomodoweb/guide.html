ORTHOMODO-WEB: Manage your Ortho Model Patients and Jobs.

IMPORTANT NOTES

The date format used is YY-MM-DD. This means that December 1st, 2017 will show as 17-12-01. This date format is the easiest format for a database and cannot get confused between day and month, e.g. is 12/7 July 12th or December 7th?

When you make a change, make sure you SAVE before moving to a different view!

OrthoModo Web is like any website: Use the REFRESH and BACK buttons to make sure your data is correct or you need to move to the previous page.

What is the Orthomodo Database system for?

Orthomodoweb is a database system for storing basic information to track orthodontic jobs. It's aim is to make it easy for many users to access and update information about the following
  1. Patient names, contact details, and Orthotrac system references, which can be easily searched
  2. Patient Scan dates and assigned Clinician
  3. Plaster Model and 3D Printed Model details
  4. Lab Items that are required, such as Retainers and Gumshields
  5. Planned Collection/Delivery type and dates

How do I use the system?

The basic process when using the system to add patient/job/lab item details is the following
  • Check if a patient already exists by searching the patient of jobs list
  • Create a New patient if he/she is not already on the system. Use the Orthotrac reference number as the patient CODE
  • Add a New JOB for that patient, or edit one of their existing jobs. A patient may have many separate jobs. A job is based on a printed 3D or plaster MODEL. If you are creating a new model, then create a new JOB
  • An UNCOLLECTED and UNFLAGGED job is OPEN: it will show up in the OPEN list. This will make it easy to find jobs that are currently in progress
  • As a JOB progresses, details must be updated, such as when a model was printed/moulded, when it is planned to be collected, etc
  • Each JOB will have an assigned MODEL USE: this will define what is to be done with a model when created. For example, Lab Items such as a Gumshield may be needed, or the clinician may need to inspect the model, or it may need to be posted directly to a dentist or patient. This should be updated if the use changes.
  • Lab Items may be added to each JOB. A Lab Item is something that is created using the Model, such as a Retainer or gumshield.
  • Each Lab Item will have a Type (e.g. Gumshield LOWER) and a Material (e.g. Essix 0.5mm, or Gumshield Blue). The Date Made can be added, and the item can be confirmed as made "Is Made?"
  • Each Lab Item will have a Type (e.g. Gumshield LOWER) and a Material (e.g. Essix 0.5mm, or Gumshield Blue). The Date Made can be added, and the item can be confirmed as made "Is Made?"
  • In order to quickly display lab item info, a Lab Item Code will be created. This will look like the following: RU 05 B M 12-01. This code is a shorthand for Retainer Upper, 0.5mm Essix, Is Blocked, Is Made, Made on 2017/12/01

Setting up the System

The system will need the following items to be added so that they can be selected for Jobs and Lab Items:
  1. Printers: 3D printers
  2. Clinicians
  3. Lab Item Types: Basic Lab Item type, such as Retainer Upper, Retainer Lower, Gumshield Upper, and Gumshield Lower.
  4. Created Model Uses*: WHat will the model be used for? Further Analysis? To create Lab Items?
  5. Lab Item Materials*: What material will a lab item such as a retainer, be created from?
  6. Model Types*: This is limited to two basic types of model: Plaster and 3D Print
    7. * These are currently only found in the ADMIN section, for which you will need to be an ADMIN user.

CSV Export

On the HOME page, you will find an option that lets you download all of the system JOBS as a CSV file. This will include a line for each Lab Item if they exist, and can be opened in Microsoft Excel. You can use this to analyse the data or create a quick backup for reference.
